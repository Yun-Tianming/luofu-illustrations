# 落浮双角色正文配图

## 核心定位

把中文内容里的关键判断、流程、结构、状态或隐喻，转成一张有认知作用的 16:9 手绘解释图。默认视觉角色是韩立与南宫婉，灵感来自《凡人修仙传》，外观以本 Skill 自带的两张角色设定图为最高优先级。

角色不是角落里的装饰，也不是每张图都要出现的情侣贴纸。根据内容选择韩立、南宫婉或二人同场，并让所选角色承担核心动作。

## 按需读取

- 任何涉及生图、改图或选角的任务，先读 `references/character-duo.md`。
- 确定画面风格时读 `references/style-dna.md`。
- 设计 shot list、结构与隐喻时读 `references/composition-patterns.md`。
- 调用图像模型前读 `references/prompt-template.md`。
- 生成后读 `references/qa-checklist.md`。
- 角色外观参考只使用 `assets/characters/han-li-character-sheet.png` 与 `assets/characters/nangong-wan-character-sheet.png`。

不要读取或使用仓库根目录 `examples/legacy-*` 中的旧“小黑”图片进行风格校准。

## 工作流

### 1. 消化正文

提炼核心观点、认知转折、适合视觉化的段落，以及不需要配图的纯文字段落。不要平均配图，优先选择核心判断、输入输出闭环、风险点、前后对比、分流、承接、探索与验证、发散与收敛等认知锚点。

### 2. 选择角色

先判断画面需要的认知功能，再选角色：

- 韩立：验证、风险、证据、复盘、规划、资源管理、止损、系统稳定、复杂决策。
- 南宫婉：灵感、探索、连接、激活、引导、变化、机会、轻盈表达、关系建立。
- 二人同场：探索与验证、速度与质量、冒险与安全、发散与收敛、行动与复盘、相互支持或交接。

默认只用完成表达所需的最少角色。只有主题本身存在互补、权衡、冲突或协作时才让二人同场。允许根据内容反转分工，不把任何一方固定为“永远正确”或“等待被救”。

### 3. 先出配图策略

用户只要求分析配图时，先输出 shot list。每张写清楚：

- 放在哪个段落后
- 图的主题与核心意思
- 结构类型
- 选角：韩立 / 南宫婉 / 双角色
- 选角理由
- 每个角色承担的动作
- 建议物件与中文标注

默认 4-8 张；短文 1-3 张；长文通常不超过 9 张。

### 4. 单张生成

用户明确要求生成时，直接用内置 `image_gen` 每张单独生成，不把多张图拼在一张里。

调用时将所选角色设定图的绝对路径传入 `referenced_image_paths`：

- 韩立单人图：只传 `han-li-character-sheet.png`。
- 南宫婉单人图：只传 `nangong-wan-character-sheet.png`。
- 双角色图：同时传两张，先韩立、后南宫婉。

不要同时使用 `num_last_images_to_include`。角色设定图只规定人物身份、服装、发型、配色和手绘质感；最终图必须是一个完整叙事场景，不能生成五视图、角色排排站、设定表或拼贴。

每张 prompt 必须包含：选角、角色动作、核心隐喻、16:9 白底手绘风格、角色一致性、简短中文标注和禁止项。二人同场时写明互动关系，不能只写“韩立和南宫婉站在画面里”。

### 5. 检查与迭代

按 `references/qa-checklist.md` 检查角色身份、选角合理性、互动能动性、构图、文字和风格。角色认错、服装互换、无故双人同框、生成设定表或人物沦为装饰时，优先重生成。

### 6. 保存交付

在 workspace 中将最终图保存到：

```text
assets/<article-slug>-illustrations/
```

按 `01-topic-name.png`、`02-topic-name.png` 顺序命名。不要覆盖已有资产，除非用户明确要求替换。

交付时报告生成数量、每张用途、选角和保存路径。
